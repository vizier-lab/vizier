# Designer Skill

This skill provides UI/UX design principles and guidelines for creating effective user interfaces.

## Core Principles

### 1. User-Centered Design
- Always consider the end user's needs and goals
- Conduct user research when possible
- Validate designs with real users
- Iterate based on feedback

### 2. Visual Hierarchy
- Use size, color, and placement to guide attention
- Most important elements should be most prominent
- Create clear focal points
- Use whitespace effectively

### 3. Consistency
- Follow established design patterns
- Maintain consistent spacing, colors, and typography
- Use a design system when available
- Keep interactions predictable

### 4. Accessibility
- Ensure sufficient color contrast (WCAG AA minimum)
- Provide alternative text for images
- Support keyboard navigation
- Design for screen readers

### 5. Simplicity
- Remove unnecessary elements
- Reduce cognitive load
- Use progressive disclosure
- Make actions clear and obvious

## Design Checklist

### Layout
- [ ] Grid system is consistent
- [ ] Spacing follows a rhythm (4px, 8px, 16px, etc.)
- [ ] Content is properly aligned
- [ ] Responsive design works on all screen sizes

### Typography
- [ ] Font hierarchy is clear (headings, body, captions)
- [ ] Line height is appropriate (1.5 for body text)
- [ ] Line length is readable (50-75 characters)
- [ ] Font sizes are readable (minimum 16px for body)

### Color
- [ ] Color palette is limited and cohesive
- [ ] Primary color is used for main actions
- [ ] Semantic colors are used consistently (success, error, warning)
- [ ] Color is not the only way to convey information

### Interactions
- [ ] Buttons look clickable
- [ ] Hover states provide feedback
- [ ] Loading states are clear
- [ ] Error messages are helpful

### Content
- [ ] Copy is clear and concise
- [ ] Labels are descriptive
- [ ] Placeholders provide examples
- [ ] Error messages explain how to fix

## Common Patterns

### Navigation
- Top navigation for main sections
- Sidebar for secondary navigation
- Breadcrumbs for deep hierarchies
- Tab navigation for related content

### Forms
- Single column for most forms
- Clear labels above inputs
- Inline validation
- Clear call-to-action

### Data Display
- Tables for structured data
- Cards for visual content
- Lists for simple items
- Charts for trends and comparisons

### Feedback
- Toast messages for quick confirmations
- Modals for important decisions
- Inline validation for forms
- Loading indicators for async operations

## Responsive Design Breakpoints

- Mobile: < 640px
- Tablet: 640px - 1024px
- Desktop: > 1024px

## Design Tokens

Use consistent design tokens for:
- Colors (primary, secondary, neutral, semantic)
- Spacing (4px base unit)
- Typography (font sizes, weights, line heights)
- Border radius (small, medium, large)
- Shadows (small, medium, large)
